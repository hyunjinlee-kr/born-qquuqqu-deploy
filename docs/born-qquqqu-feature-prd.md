## 1. 제품 개요

### 1.1 제품 정의

Born ★ qquqqu는 모바일 웹 브라우저에서 인생네컷 스타일의 포토스트립을 촬영·저장할 수 있는 빈티지 포토부스 웹앱이다. 앱 설치 없이 URL 접속만으로 사용 가능하며, 사진 또는 3초 영상 4개를 촬영해 커스텀 프레임·배경이 적용된 스트립을 즉시 생성한다.

### 1.2 핵심 가치

- **앱 설치 불필요** — 모바일 브라우저 즉시 실행
- **인생네컷 감성** — 빈티지 프레임 + 커스텀 배경
- **사진 & 영상 모두 지원** — 정지사진 4장 또는 3초 영상 4클립
- **빠른 경험** — 선택 → 촬영 → 저장 1분 이내
- **어드민 확장** — PNG 프레임 디자인을 자유롭게 추가/관리

### 1.3 타겟 사용자

| 세그먼트 | 특징 | 주요 Use Case |
|----------|------|---------------|
| 10~20대 여성 | 인생네컷 문화 친숙, SNS 공유 적극적 | 친구와 촬영 후 즉시 공유 |
| 커플 | 기념일 사진/영상 | 2×2 그리드 네컷 생성 |
| 이벤트 주최자 | 팝업, 파티 | 브랜드 프레임 포토부스 운영 |

## 2. 전체 사용자 플로우

```
[시작 화면]
    ↓ Enter
[레이아웃 선택]  — 1×4 세로 / 2×2 그리드
    ↓
[프레임 선택]    — 색상 6종 + 패턴 3종 + PNG 커스텀
    ↓
[배경 선택]      — 별 / 하트 / 단색 / 없음 / 업로드
    ↓
[사진 등록 방법 선택]
    ├─ 📷 직접 촬영 ─→ [카메라 화면]
    │                      ↕ 모드 전환
    │                   📷 사진 모드   🎬 영상 3초 모드
    │                   (4장 순차)    (3초 × 4클립)
    └─ 🖼️ 앨범 선택 ─→ 갤러리에서 4장 선택
    ↓
[결과 화면]
    ├─ 저장하기 (이미지 JPEG / 영상 클립 webm·mp4)
    ├─ 공유하기 (Web Share API)
    ├─ ← 다시 찍기
    └─ 처음으로
```

### F-03. 배경 선택

| 항목 | 내용 |
|------|------|
| 기능 ID | F-03 |
| 옵션 | 별 패턴 6종 / 하트 패턴 6종 / 단색 6종 / 없음 / 직접 업로드 |

### F-04. 사진 등록 방법 선택

| 항목 | 내용 |
|------|------|
| 기능 ID | F-04 |
| 방법 1 | 직접 촬영 → 카메라 화면으로 이동 |
| 방법 2 | 앨범 선택 → `input[type=file]` 갤러리 직접 호출 |
| 앨범 검증 | 정확히 4장 선택 필수, 미충족 시 안내 |
| 앨범 로딩 | 처리 중 전체화면 오버레이 표시 |

### F-05. 카메라 촬영 — 사진 모드

| 항목 | 내용 |
|------|------|
| 기능 ID | F-05-A |
| 카운트다운 | 3초, 카메라 뷰 내부 오버레이 (블러 없음) |
| 플래시 | 흰 오버레이 fade-out 효과 |
| 필터 | B&W / Y2K / 기본 (실시간 선택) |
| 진행 표시 | 4개 프로그레스 도트 |
| 미러링 | 전면 카메라 좌우 반전 적용 |
| 저장 형식 | JPEG dataURL → 포토스트립 캔버스에 합성 |

### F-06. 카메라 촬영 — 영상 모드 *(신규)*

| 항목 | 내용 |
|------|------|
| 기능 ID | F-05-B |
| 진입 | 카메라 화면 상단 "🎬 영상 3초" 탭 선택 |
| 촬영 방식 | ● 버튼 탭 → 3초 카운트다운 → 자동 녹화 시작 → 3초 후 자동 종료 |
| 녹화 표시 | 카메라 뷰 좌상단 🔴 REC + 초 카운트 |
| 반복 | 4클립 녹화 완료까지 반복 |
| 필터 | 영상 모드에서 필터 탭 미노출 (미지원) |
| 저장 형식 | Blob (webm / mp4) |
| 출력 | 결과 화면에서 선택 레이아웃에 맞게 4개 영상 루프 재생 |

### F-07. 카메라 권한 안내 *(신규)*

| 에러 상황 | 안내 내용 |
|-----------|-----------|
| 권한 거부 | 주소창 자물쇠 → 카메라 허용 방법 단계별 안내 |
| 카메라 없음 | 기기 카메라 연결 확인 요청 |
| 다른 앱 사용 중 | 다른 앱 종료 후 재시도 |
| HTTP 환경 | HTTPS/localhost 접속 안내 |
| 미지원 브라우저 | Chrome 최신 버전 권장 |

→ 카메라 뷰 위 오버레이로 표시 + **다시 시도** 버튼 제공

### F-08. 결과 화면

| 항목 | 내용 |
|------|------|
| 기능 ID | F-06 |
| 사진 모드 | 포토스트립 이미지 (JPEG) 한 화면 fit 표시 |
| 영상 모드 | 4개 영상 클립을 선택 레이아웃에 맞게 루프 재생 |

## 4. 어드민 기능

### 4.1 어드민 접근

| 항목 | 내용 |
|------|------|
| URL | /admin |
| 인증 | 이메일 + 비밀번호 로그인 → JWT 발급 |
| 권한 | super: 전체 권한 / editor: 등록·수정만 가능 |

### 4.2 프레임 관리

| 기능 | 설명 |
|------|------|
| 목록 조회 | 전체 프레임 목록 (활성/비활성 포함) |
| 프레임 등록 | 이름 입력 + 1×4 PNG + 2×2 PNG 업로드 |
| 미리보기 | 업로드된 PNG 썸네일 즉시 확인 |
| 활성/비활성 | 토글 스위치로 사용자 노출 여부 제어 |
| 정렬 순서 | 숫자 입력으로 사용자 화면 노출 순서 조정 |
| 삭제 | 등록된 프레임 제거 (S3 파일 + DB 레코드) |

## 6. 비기능 요구사항

### 6.1 사용성
- 전체 플로우 1분 이내 완료
- 각 단계 뒤로가기 제공
- 한 화면에 핵심 액션 1개 원칙

### 6.2 접근성
- 터치 타겟 최소 44 × 44px
- 색상 대비 WCAG AA 기준
- 카메라/갤러리 권한 에러 시 명확한 안내 + 재시도 버튼

### 6.3 환경 요구사항
- **카메라/영상 기능**: HTTPS 또는 localhost 환경 필수
- **오프라인**: Phase 1 미지원 (Phase 2 이후 PWA 검토)
